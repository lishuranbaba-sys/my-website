# 织影 AI 官方网站

织影 AI 是一款面向普通用户、内容创作者、商家、企业及行业从业者的 AI 内容创作工具。

平台聚合文字、图片和视频生成能力，帮助用户降低 AI 工具的使用成本和操作门槛，让不懂提示词、没有设计基础、不会剪辑和写文案的用户，也能够快速完成内容生产。

本仓库用于存放织影 AI 官方网站的前端项目代码。

---

## 项目信息

- 项目名称：织影 AI 官方网站
- GitHub 仓库名称：`my-website`
- 项目目录：`zhiying-ai-website`
- 前端框架：React
- 构建工具：Vite
- 样式框架：Tailwind CSS
- 动画框架：Framer Motion
- 部署方式：GitHub Pages、EdgeOne Pages、Vercel
- 网站类型：响应式产品展示官网
- 支持设备：电脑、平板、手机

## 产品定位

织影 AI 希望解决普通用户在使用 AI 工具时遇到的以下问题：

1. 海外 AI 产品订阅费用较高
2. 文字、图片和视频功能分散在不同平台
3. 部分工具需要特殊网络环境
4. 英文界面和复杂参数增加使用门槛
5. 普通用户不会编写专业提示词
6. 商家制作图片、文案和视频素材成本较高
7. 内容创作者不知道应该拍什么、写什么
8. 中小企业缺少专业设计和内容团队

织影 AI 将常用的文字、图片和视频创作能力整合到一个平台中，让用户能够用更简单的方式完成内容创作。

织影 AI 的核心理念是：

> 聚合国际顶级模型，让普通人也用得起 AI 创作。

## GitHub Pages 部署

当前 GitHub 仓库名称：

```text
my-website
```

项目文件位于：

```text
zhiying-ai-website
```

预计 GitHub Pages 地址：

```text
https://lishuranbaba-sys.github.io/my-website/
```

最终网站地址以 GitHub 仓库中的以下页面为准：

```text
Settings → Pages
```

### 开启 GitHub Pages

进入 GitHub 仓库后：

1. 点击顶部 `Settings`
2. 点击左侧 `Pages`
3. 找到 `Build and deployment`
4. 将 `Source` 设置为 `GitHub Actions`
5. 保存设置

### 自动部署工作流

在仓库最外层创建：

```text
.github/workflows/deploy.yml
```

文件内容：

```yaml
name: Deploy Zhiying AI to GitHub Pages

on:
  push:
    branches:
      - main
  workflow_dispatch:

permissions:
  contents: read
  pages: write
  id-token: write

concurrency:
  group: pages
  cancel-in-progress: true

jobs:
  build:
    runs-on: ubuntu-latest

    steps:
      - name: Checkout repository
        uses: actions/checkout@v4

      - name: Setup Node.js
        uses: actions/setup-node@v4
        with:
          node-version: 20

      - name: Configure GitHub Pages
        uses: actions/configure-pages@v5

      - name: Install dependencies
        working-directory: ./zhiying-ai-website
        run: npm install

      - name: Build website
        working-directory: ./zhiying-ai-website
        run: npm run build

      - name: Upload website files
        uses: actions/upload-pages-artifact@v3
        with:
          path: ./zhiying-ai-website/dist

  deploy:
    environment:
      name: github-pages
      url: ${{ steps.deployment.outputs.page_url }}

    runs-on: ubuntu-latest
    needs: build

    steps:
      - name: Deploy website
        id: deployment
        uses: actions/deploy-pages@v4
```

提交该文件后，打开仓库顶部的：

```text
Actions
```

查看部署状态。

状态说明：

```text
黄色圆点：正在构建
绿色对勾：部署成功
红色叉号：部署失败
```

部署成功后，进入：

```text
Settings → Pages
```

即可看到网站地址。

## Logo 路径说明

部署到 GitHub Pages 后，网站不在域名根目录，而是在：

```text
/my-website/
```

为了保证 Logo 能够在本地、GitHub Pages、Vercel 和 EdgeOne 中正常显示，建议在代码中使用：

```js
const LOGO_IMAGE_SRC = `${import.meta.env.BASE_URL}logo.png`;
```

不要长期使用：

```js
const LOGO_IMAGE_SRC = "/logo.png";
```

因为 `/logo.png` 会从域名根目录读取，在 GitHub Pages 项目地址中可能出现图片无法显示的问题。

## 常见问题

### GitHub Pages 打开后是空白页

检查：

```text
zhiying-ai-website/vite.config.js
```

确认包含：

```js
base: "/my-website/",
```

`my-website` 必须与 GitHub 地址栏中的仓库名称完全一致。

### Logo 无法显示

确认 Logo 文件存在：

```text
zhiying-ai-website/public/logo.png
```

建议 Logo 路径使用：

```js
const LOGO_IMAGE_SRC = `${import.meta.env.BASE_URL}logo.png`;
```

### GitHub Actions 安装失败

检查：

- `package.json` 是否存在
- `package.json` 是否位于 `zhiying-ai-website` 文件夹
- 工作流中的 `working-directory` 是否正确
- Node.js 版本是否为 20
- 项目依赖名称是否正确

### GitHub Actions 构建失败

在电脑本地运行：

```bash
cd zhiying-ai-website
npm install
npm run build
```

如果本地也报错，需要先修复代码错误。

### 网站修改后没有更新

检查：

1. 修改是否成功提交
2. Actions 是否生成新的运行记录
3. Actions 是否显示绿色对勾
4. 浏览器是否使用旧缓存
5. 是否打开了正确的网站地址

强制刷新方式：

Windows：

```text
Ctrl + F5
```

Mac：

```text
Command + Shift + R
```

### GitHub 网页无法编辑文件

可以尝试：

1. 关闭浏览器网页翻译
2. 刷新 GitHub 页面
3. 重新进入文件编辑
4. 按键盘英文句号 `.` 打开 `github.dev`
5. 在在线编辑器中修改并提交

### 手机端页面过长

手机端已经通过以下方式优化：

- 隐藏部分电脑端展示模块
- 模型卡片横向滑动
- 会员卡片横向滑动
- 减少模块上下间距
- 缩小标题字号
- 增加快捷导航

如果仍需缩短，可在 `App.jsx` 中使用 Tailwind 的：

```text
hidden md:block
```

让部分模块只在平板和电脑端显示。
